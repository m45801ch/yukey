## 專案概述

**聲聲慢 (SpeakSlow)** 是一款專注於中文語音轉文字的跨平台應用程式，採用自研的 Sherpa-ONNX 離線辨識引擎，結合 AI 文字優化技術。

### 品牌定位

- **名稱**：聲聲慢 (SpeakSlow)
- **Slogan**：「讓每一個字，都被聽見」
- **核心價值**：高準確、低延遲、隱私保護、離線可用

### 目前技術資產

| 資產 | 狀態 | 說明 |
|------|------|------|
| Sherpa-ONNX 離線辨識 | ✅ 完成 | Paraformer + Zipformer 雙模型 |
| 串流即時辨識 | ✅ 完成 | Zipformer Bilingual |
| 熱詞功能 | ✅ 完成 | 辨識時權重提升 |
| Silero VAD | ✅ 完成 | 跳過靜音段 |
| 字典後處理 | ✅ 完成 | 專有名詞校正 |
| DeepSeek AI 優化 | ✅ 完成 | 文字潤飾 |
| 標點恢復 | ✅ 完成 | FunASR ct-punc |

## Phase B：網頁版開發

### B.1 目標

在 ququ 專案內建立網頁版，直接使用現有的 sherpa_server.py 後端。

> **決策**：原 AprilVoice 專案已廢棄，其「雙向顯示」功能將作為聲聲慢網頁版的特色功能之一。

### B.2 架構

```
聲聲慢 Web 版架構：
┌─────────────────────────────────────────────────────┐
│                    ququ 專案                         │
├─────────────────────────────────────────────────────┤
│                                                     │
│  ┌─────────────┐         ┌─────────────┐           │
│  │  web/       │  HTTP   │ sherpa_     │           │
│  │  (新增)     │ ◄─────► │ server.py   │           │
│  │  React SPA  │  WS     │ (現有)      │           │
│  └─────────────┘         └─────────────┘           │
│                                                     │
└─────────────────────────────────────────────────────┘
```

### B.3 網頁版功能

| 功能 | 說明 | 優先級 |
|------|------|--------|
| 基本語音辨識 | 錄音 → 辨識 → 顯示 | 🔴 高 |
| 串流顯示 | 邊說邊顯示（Zipformer） | 🔴 高 |
| 雙向顯示模式 | 給對面的人看（原 AprilVoice 功能） | 🟡 中 |
| AI 文字優化 | DeepSeek 潤飾 | 🟡 中 |
| 歷史記錄 | LocalStorage 暫存 | 🟢 低 |

### B.4 開發任務

| # | 任務 | 說明 | 狀態 |
|---|------|------|------|
| B.1 | 建立 web/ 目錄 | Vite + React + TypeScript | ⏳ |
| B.2 | 擴充 sherpa_server.py | 新增 HTTP API + CORS | ⏳ |
| B.3 | 實作錄音元件 | Web Audio API | ⏳ |
| B.4 | WebSocket 串流 | 即時辨識顯示 | ⏳ |
| B.5 | 雙向顯示頁面 | 180度翻轉文字 | ⏳ |
| B.6 | Docker 配置 | 前後端容器化 | ⏳ |
| B.7 | 部署上線 | Vercel + GPU Server | ⏳ |

### B.4 API 設計

```
POST   /api/transcribe          # 單次辨識（上傳音檔）
WS     /ws/stream               # 串流辨識（即時）
POST   /api/hotwords            # 設定熱詞
GET    /api/hotwords            # 取得熱詞
POST   /api/dictionary          # 新增字典
GET    /api/dictionary          # 取得字典
POST   /api/optimize            # AI 文字優化
GET    /api/health              # 健康檢查
```

### B.5 部署架構

