## 測試環境
- 平台: Windows 10/11
- CPU: 4 核心

## 性能對比

| 指標 | sherpa-onnx | FunASR (streaming) | 差異 |
|------|-------------|-------------------|------|
| **模型載入時間** | 3.25 秒 | 15.16 秒 | 🟢 快 4.7x |
| **總處理時間** | 1.75 秒 | 20.71 秒 | 🟢 快 11.8x |
| **平均 RTF** | 0.037 | 0.440 | 🟢 快 11.9x |
| **實時倍數** | 26.8x | 2.3x | 🟢 快 11.7x |
| **記憶體使用** | ~547 MB | ~2+ GB (估計) | 🟢 省 ~75% |
| **模型大小** | ~223 MB (int8) | ~944 MB | 🟢 小 ~75% |

## 總結

### sherpa-onnx 優勢 ✅
1. **速度快 11-12 倍** - 使用 ONNX Runtime 優化
2. **記憶體省 75%** - 約 547MB vs 2+GB
3. **模型體積小 75%** - 223MB vs 944MB
4. **識別準確度更高** - 特別是對於標準普通話
5. **更少的重複字/錯字** - 輸出更乾淨
6. **不需要 PyTorch** - 減少依賴

### FunASR 優勢 ⚠️
1. **更完整的功能** - 支援 VAD、標點等
2. **官方維護** - 阿里巴巴官方支援
3. **更多模型選擇** - 支援多種模型

### 建議
**強烈建議採用 sherpa-onnx 作為新的 ASR 引擎**，原因：
- 性能提升顯著（10x+）
- 打包體積大幅縮小
- 識別準確度更好
- 部署更簡單（無需 PyTorch）

### 遷移注意事項
1. 需要下載 sherpa-onnx 的 Paraformer 模型
2. API 調用方式不同，需要修改服務端代碼
3. 可以考慮保留 FunASR 作為備選方案
