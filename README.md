# yukey — 智慧語音輸入與 AI 文本修飾助理

一個基於 Tauri 2.x、Rust 後端與 React 前端所建構的跨平台桌面語音助手。

## 🌟 主要功能特點 (Key Features)

### 1. 🎙️ 語音即時錄製與精準辨識
* **全域快捷鍵啟動**：支援自訂鍵盤快速鍵，隨時開啟/關閉錄音，並支援 PTT (按住說話) 模式。
* **智慧語音活動偵測 (VAD)**：內建 Silero VAD 技術，自動過濾背景雜音與空白無聲片段。
* **本地與雲端語音轉文字 (STT)**：支援本地 Whisper 模型推理與 Parakeet 引擎，確保離線狀態下的隱私安全。

### 2. 🤖 AI 語音文本修飾 (Prompt Plugin)
* **多樣化修飾模式**：預設內建一般修飾、精準逐字稿、聊天日常、商務會議等模式，一鍵快速潤飾轉錄文字。
* **自訂修飾指令**：允許使用者自由建立自訂 AI 提示詞指令，讓輸出文本更貼合個人語氣。
* **本地熱詞 (Hotwords)**：支援設定本機優先辨識字詞，防止縮寫或專有名詞被誤判。
* **專業領域詞庫**：針對程式設計、法律、醫學、工程等專業領域，載入專用語法與專有名詞。

### 3. 🎯 語音糾錯對照表 (Auto Correction)
* **規則表格對照**：提供視覺化的「原始錯字/音 -> 正確文字」規則管理，動態將對照規則注入 AI 進行前置處理。
* **動態語音修復**：自動校正特定發音不清或辨識常出錯的常見字。

### 4. 💾 一鍵 ZIP 備份與還原
* **完整設定打包**：將所有的自訂修飾模式、詞庫、熱詞、糾錯規則一鍵壓縮導出為 ZIP 備份檔（內含 JSON、CSV、TXT 與可讀性 Markdown 報告）。
* **跨裝置無縫移轉**：支援導入 ZIP 檔案，立即還原所有設定與規則，方便多台電腦同步。

### 5. 📂 歷史記錄與管理
* **轉錄歷程檢索**：保留每次語音轉錄歷史，支援快速搜尋、儲存或刪除。
* **二次辨識 (Retry)**：如果對轉錄結果不滿意，可以隨時對歷史語音重新進行 STT 與 AI 修飾。
* **靈活空間配置**：可設定語音保留期限，自動清理老舊錄音檔案。

### 6. 🎨 Premium 質感與多主題切換
* **多款極致主題**：內建「禪意自然 (Zen Natural)」、「科技深黑 (Dark Tech)」與「極致精緻白 (Premium Light)」等精心調配的設計主題。
* **深淺色自適應**：UI 介面完全支持深色模式與淺色模式，操作體驗流暢 premium。

## 🚀 使用指南 (Usage Guide)

### 1. 首次啟動設定
* 首次執行程式時，會進入引導畫面（Onboarding）。
* 請選擇並下載一個語音辨識模型（例如 `Small (70MB)` 可快速下載且佔用記憶體小，適合作為一般辨識；若追求極致準確可下載較大的模型）。
* 下載完成後即會進入主畫面與系統背景執行（常駐於系統右下角托盤區）。

### 2. 開始語音輸入
* **錄音啟動**：在任何文字輸入框中（例如瀏覽器、Word、LINE 等），按下全域快捷鍵（預設為自訂快速鍵）即可開始說話。
* **狀態預覽**：錄音時，螢幕邊緣會顯示懸浮指示器，代表正在接收音量。
* **完成轉錄**：說話結束後再次按下快捷鍵，程式會自動完成「語音辨識 ➔ AI 智慧潤飾 ➔ 自動粘貼」的流暢步驟，直接將修飾後的文字輸入到你剛剛的游標處。
* **取消操作**：錄音中如果想取消，可按自訂的取消快捷鍵（預設為 `Esc`）。

### 3. AI 提示詞修飾與詞庫
* **切換模式**：在設定的「Overview (概覽)」分頁中，可以切換不同的「修飾模式」（例如需要完整保留發音字眼選「逐字稿」，需要精簡語氣選「簡明模式」）。
* **啟用專業詞庫**：若要進行程式設計、法律或醫學等專業內容輸入，可以勾選對應的「專業領域詞庫」來提升術語精準度。
* **一鍵備份**：當你在新電腦安裝時，可透過設定介面一鍵導出為一個 ZIP 備份檔，並在新設備上還原。

## 🛠️ 開發、打包與建置說明 (Development & Build)

### 📌 前置準備 (Prerequisites)
建置前請確保您的電腦上已安裝以下環境：
* **[Rust](https://rustup.rs/)** (最新穩定版本)
* **[Bun](https://bun.sh/)** (專案的套件管理器)

### 📥 1. 安裝專案依賴
```bash
bun install
```

### 💻 2. 本機開發偵錯模式 (Tauri Dev)
執行以下指令會同時啟動前端 Vite 伺服器以及 Tauri 的桌面視窗，適合開發時實時預覽變更：
```bash
bun run tauri dev
```

### 📦 3. 打包專案 (Build Production)
使用以下命令會編譯前端程式碼並建置成 Release 模式下的**安裝檔與獨立執行檔**。
* 編譯後的打包檔案將放置在：`src-tauri/target/release/bundle/`
* 在 Windows 系統上會產出 `.exe` 安裝檔與安裝套件。

```bash
bun run tauri build
```
> [!TIP]
> 預設的打包程序會自動執行 `tsconfig` 檢查與前端的 `vite build` 構建，請確保在打包前沒有 TypeScript 語法錯誤。
